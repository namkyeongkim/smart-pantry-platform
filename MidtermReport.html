1. Pantry Manager

1.1. API

Interface to communicate with Spoonacular API and our PostgreSQL hosted on Neon. Integration layer for Administrator features, adding/removing ingredients from the users pantry, adding new recipes to local storage.

1.1.4. Favorites

favorites (
    id SERIAL PRIMARY KEY,
    user_id INTEGER NOT NULL REFERENCES users(user_id) ON DELETE CASCADE,
    recipe_id INTEGER NOT NULL,
    title TEXT NOT NULL,
    image TEXT,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    CONSTRAINT favorites_user_recipe_unique UNIQUE (user_id, recipe_id)
);

Users can store there favorite recipes locally in a shared library allowing for uninterrupted access without an internet connection. The favorites can be shared among the users group. Recipes can be modified to customize dishes to the users tastes.

1.2. Client

A mobile application will allow the user to easily search for a recipe based on their tastes or nutritional requirements. A shareable key will allow the user to invite other app users to a shared family account that will provide access to a group pantry and recipe library. Users can use their phones camera or a webcam to look up items by their UPC codes and easily add/remove items from the pantry.

1.4. Project Outline

A project to create a mobile/web application that will provide a user recipe recommendations based on user tastes and dietary preferences. The user's account can be used by multiple users creating a shared family group that gives the primary user the ability to track what everyone is eating. Recipes are suggested based on user responses to basic questions about their tastes and current desire. The recipes can be provided from the users local library or searched directly from the web. The recipes can be listed in ranked order based on the ingredients currently in the users pantry. With recipes that require fewer purchased items being ranked higher. Ingredients can be automatically deducted from the pantry based on the recipes requirements. Shopping lists can be generated to gather any items that the user is currently missing. Using their phone to scan the UPC of ingredients to add them to the pantry and remove them from the grocery list. Meals plans can be generated from the recipe library or searched with basic search functionality. Primary user can see the diets of the other group users. Using the stored dietary and nutritional information to create basic reports and graphs of the families eating and cooking habits.
